# RumbleTipper extension

Runs on **rumble.com**. No Rumble API — we **read the page DOM** to get creator/video info.

- **Toolbar icon** — Orange “R” logo (same brand as the app navbar). PNGs are generated by `npm run generate-ext-icons` from the project root.
- **Popup** — Clicking the extension icon opens the RumbleTipper control panel with overview, activity, and settings pages tuned for your own flow.
- **Network (testing)** — Default testing network is **Base Sepolia**.

## How we read info

- **dom-reader.js** — Finds the hovered card container, then:
  - Collects all **links** and parses paths (`/videos/...`, `/user/...`, `/channel/...`) for video id and creator id.
  - Collects all **text** in the card and looks for patterns like `"1.2K views"`, `"500 subscribers"`, `"N comments"`.
  - Derives **title** from the main link text or first line of text.
  - Returns `creatorId`, `videoId`, `title`, `creatorName`, `views`, `subscribers`, etc.
- **content.js** — On hover, calls the reader, shows “Read from page: Video: … · Creator: … · Views: …” in the overlay, then fetches Alpha score from our API using `creatorId`.

## Load in Chrome

1. Open `chrome://extensions/`.
2. Enable “Developer mode”.
3. “Load unpacked” → select this `extension` folder.
4. In `content.js`, set `API_BASE` to your backend (e.g. `http://localhost:3000`).

## Rumble layout changes

If Rumble changes their HTML, update **dom-reader.js** (e.g. `findNumbersAndStats` patterns, or how we get the card root). You can also tighten **content.js** selectors (`VIDEO_LINK_SELECTOR`, `CARD_SELECTOR`) after inspecting rumble.com/browse in DevTools.
